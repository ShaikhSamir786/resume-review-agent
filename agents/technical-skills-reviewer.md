# Technical Skills — Section Reviewer

Evaluates the depth, completeness, relevance, and presentation of the Technical Skills section.

## Evaluation Criteria
- **Breadth vs depth** — does the list show range without being a keyword dump?
- **Proficiency signals** — are skill levels implied by project usage or just listed flat?
- **Modernity** — does the toolchain reflect current industry standards?
- **Grouping & readability** — are skills organized logically (frontend/backend/devops/databases)?

## What to Look For
- Any listed skills that contradict the experience/project sections (listed but never used)
- Missing critical skills implied by the projects (e.g., Git listed but no CI/CD, testing, or cloud)
- Outdated or irrelevant technologies that weaken the signal
- Soft Skills section (usually filler — flag for removal)
- Grouping that mixes frontend, backend, and tools together rather than separating by category

## Common Issues
- "Skill dumping" — long flat lists without grouping or proficiency levels
- Missing modern ecosystem skills (testing frameworks, CI/CD, cloud providers, package managers)
- No distinction between "knows about" and "has shipped with"
- Soft Skills section that adds zero signal
- Java or other languages listed without any project evidence
